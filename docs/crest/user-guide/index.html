使用总览

面向业务用户和数据分析人员的 Crest 使用入口，覆盖登录、数据源、数据集、图表、仪表盘、大屏、分享、导出和血缘。

本章面向日常使用 Crest 的业务分析人员、数据开发人员、报表制作人员、部门管理员和项目交付人员。内容按完整操作链路组织，可从登录和数据门户开始逐步阅读，也可直接进入正在处理的功能页。

进入系统后，普通用户通常先进入数据门户，查看已发布并授权给自己的仪表盘和数据大屏；具备后台权限的用户可进入工作台，继续创建、编辑和管理资源。工作台会展示资源统计、快速创建、收藏和最近使用，是分析人员和管理员每天进入后台后的起点。

截图中的顶部菜单决定当前账号可进入的模块；左侧卡片可快速查看当前账号可见的仪表盘、数据大屏和数据集数量；中间区域适合从收藏或最近使用继续工作。如果实际菜单和截图不同，通常是权限差异，不能说明系统异常。

一条完整分析链路

开始前先确认权限

不同用户看到的菜单不同，这是正常现象。Crest 的菜单、资源和数据范围都受权限控制。

目标任务需要确认什么
只查看仪表盘或大屏是否有对应资源查看权限
新建仪表盘或大屏是否有仪表盘或数据大屏菜单权限、创建权限
新建数据集是否有数据集菜单权限、数据源访问权限
新建数据源是否有数据源菜单权限，以及数据库账号信息
配置分享是否允许开启分享，以及分享策略是否被管理员限制
导出文件是否有导出权限，数据量是否符合导出限制

如看不到手册中的某个入口，不要反复刷新页面。请把“账号、缺少的菜单、资源名称、截图”发给管理员，由管理员检查角色、菜单权限和资源权限。

核心概念

概念说明操作时怎么理解
数据源Crest 读取数据的位置数据库、Excel、API 等连接
数据集图表使用的数据模型对字段、口径、权限、缓存做统一管理
维度用来分组、筛选、下钻的字段日期、地区、组织、产品、客户
指标用来计算和统计的数值字段销售额、订单数、毛利率、工单量
图表单个可视化组件指标卡、柱状图、折线图、表格
查询组件页面上的筛选器时间、地区、组织、项目、状态
仪表盘日常分析页面适合筛选、查看、导出和复盘
数据大屏全屏展示页面适合会议室、展厅、值班监控
数据门户普通用户查看已发布资源的入口收藏、最近使用、搜索、打开授权资源
数据血缘上下游依赖关系删除、改字段、改数据集前先看影响

推荐学习顺序

登录、数据门户与工作台

先熟悉登录入口、门户资源、收藏、最近使用、后台工作台和账号菜单。

数据源

学会创建数据库、Excel 或 API 数据源，能测试连接和查看表字段。

数据集

学会选择表、写 SQL、配置字段、计算字段、权限和缓存。

图表

学会选择图表类型，拖拽维度指标，配置过滤、排序、样式和高级交互。

仪表盘和数据大屏

学会把图表、筛选器、文本、图片和其他组件组装成可交付页面。

分享、导出和血缘

学会把结果交付出去，并在变更前检查影响范围。

保存、发布、分享的区别

操作含义什么时候用
保存保存当前编辑内容编辑过程中随时保存
发布将当前版本变成可正式查看版本页面确认可交付后
预览临时查看页面效果发布前检查布局和数据
门户在登录态下查看已授权的已发布资源普通用户日常访问
分享生成外部访问链接给没有编辑权限的人查看
导出生成文件汇报、归档或离线处理

分享和导出都属于交付动作。分享适合让别人在线查看，导出适合汇报、归档和离线处理。对外分享前应确认数据是否脱敏、链接是否设置有效期、访问人是否符合授权范围。

交付前总检查

  1. 页面标题是否清楚。
  2. 数据源连接是否稳定。
  3. 数据集字段口径是否明确。
  4. 图表维度和指标是否配置正确。
  5. 筛选器是否能影响目标图表。
  6. 普通用户权限是否验证过。
  7. 分享链接是否设置有效期和密码。
  8. 导出文件是否能下载且内容正确。
  9. 数据血缘中是否能看到关键依赖。
