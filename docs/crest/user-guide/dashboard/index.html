仪表盘

Crest 仪表盘操作手册，覆盖列表管理、画布编辑、图表配置、数据集字段、查询组件、联动、跳转、预览发布和排错。

仪表盘用于日常分析和经营看板。它适合放在工作台、部门例会、业务复盘和管理驾驶舱中反复查看。一个合格的仪表盘不仅要能展示图表，还要让用户能筛选、对比、下钻并理解指标口径。

入口：顶部菜单选择“仪表盘”。

列表页用于创建、查找和维护仪表盘。新建页面前选择文件夹；查看页面前确认是否已发布；调整他人维护的页面前，建议先复制副本再编辑。仪表盘通常会长期被业务人员访问，命名和目录要比临时分析更规范。

示例说明

本章以 Crest 内置演示数据说明仪表盘做法。截图中的字段和页面名称用于示例，实际项目应替换为自己的数据集、业务指标和权限范围。

先认识页面

仪表盘列表用于管理文件夹、仪表盘资源和收藏。正式项目中建议按业务主题建立文件夹，不应将所有页面都放在根目录。

区域作用使用建议
左侧资源树组织文件夹和仪表盘按部门、业务域或项目分组
列表区域查看仪表盘名称、类型、创建人、更新时间用更新时间判断页面是否仍在维护
新建入口创建仪表盘或文件夹新建前先确认数据集已经准备好
收藏/最近使用快速访问常用页面常用经营看板建议收藏
操作列编辑、复制、移动、删除、预览、发布等删除前确认是否被分享或嵌入

创建仪表盘前先准备三件事：

  1. 已经能正常预览的数据集。
  2. 明确的业务问题，例如“看每日销售走势和区域贡献”。
  3. 页面使用对象，例如管理层、运营人员、项目负责人或外部客户。

进入编辑器

新建仪表盘后会进入编辑器。初始画布为空，需要先添加图表、查询组件、文本或媒体组件。

编辑器画布中间是页面主体，顶部是组件入口，右侧是页面或组件配置。新建页面时先设置整体主题和画布，再添加图表；选中图表后，右侧会从“仪表盘配置”切换成“图表配置”。

编辑器区域说明：

区域作用操作重点
顶部左侧返回、页面名称、撤销、重做修改前先确认正在编辑正确页面
顶部组件区图表、查询组件、富文本、媒体、Tab、更多、复用所有组件都从这里添加
中间画布放置和排版组件拖动组件调整位置，拖边调整大小
右侧配置面板配置选中组件的数据、样式和高级选项选中不同组件时内容会变化
顶部右侧外部参数、批量操作、预览、保存、发布保存后再做预览和发布

建议新建后先保存一次。保存后页面有稳定 ID，后续配置联动、跳转、外部参数时更可靠。

添加第一个图表

点击顶部“图表”后，会展开图表类型面板。不同图表适合不同分析目标。

图表面板里每一种图表都对应一种分析问题。首次制作仪表盘时，建议优先使用指标卡、折线图、柱状图和明细表，字段和口径熟悉后再使用双轴、漏斗、矩阵等复杂图表。

第一次制作仪表盘时，可以先按“四类基础图表”练习：指标卡看当前值，折线图看趋势，柱状图看对比，表格看明细。掌握这四类组件后，大多数经营分析页面都可以完成基础搭建。

常用图表选择建议：

图表类型适合展示典型字段
指标卡核心数字、目标达成、当前状态销售额、订单数、完成率
表格/明细表明细记录、排行、台账门店、客户、项目、金额
折线图/面积图时间趋势日期 + 销售额、订单数
柱状图/条形图分类对比、排名区域/产品 + 指标
饼图/环形图占比结构分类 + 指标
双轴图两个指标同趋势对比日期 + 销售额 + 订单数

选择图表类型

从图表面板选择一个图表。首次配置可从指标卡、折线图、柱状图和表格开始。

放到画布

点击或拖拽图表到画布。放入后先调整到合适大小，再配置样式。

选择数据集

在右侧数据面板选择数据集。看不到数据集时，检查数据集权限和所在文件夹。

拖拽字段

把维度和指标拖到图表所需位置。指标卡通常只需要一个指标；趋势图需要日期维度和指标；柱状图需要分类维度和指标。

刷新图表

字段放好后刷新或更新图表，确认画布上能显示数据。

再调样式

数据正确后，再调整标题、颜色、标签、图例、坐标轴和数值格式。

选择数据集时，优先选择已经通过预览和口径确认的数据集。正式页面不应直接使用临时测试数据集。

数据集选择面板用于给当前图表绑定数据来源。选择前先确认数据集名称和目录，避免选到同名测试数据集。绑定后，当前图表只会显示这个数据集里的字段。

选择数据集时建议看三点：名称是否是正式数据集，创建人或最近更新时间是否可信，字段是否覆盖当前图表需要的维度和指标。如果页面里多个图表共用同一个数据集，后续查询组件和联动会更容易配置。

选择数据集后，右侧会展示字段。字段通常分为维度和指标。拖字段时要根据图表类型配置，文本字段不应放入指标区，金额字段不应放入维度区。

字段列表中，文本、日期、组织、状态通常作为维度；金额、数量、时长、比率通常作为指标。拖拽字段前，先确认字段名称和业务口径，尤其是“订单数”“需求数”“销售额”这类核心指标。

字段分类错误是常见问题。例如“项目名称”应放入维度，“需求数”应放入指标；“日期”适合放 X 轴，“负责人”适合做筛选或明细列。放字段前要先判断：这个字段用于分组，还是用于计算？

下图展示了把“销售额”放入指标卡后的状态。右侧字段区出现已使用字段，画布中的组件会根据字段刷新。

字段放入后，先确认图表是否正常渲染，再检查数值是否合理。指标卡没有维度时通常显示汇总值；如果数值明显过大，先检查聚合方式和数据集关联关系。

图表渲染成功并不等同于配置正确。保存前至少核对一次数据集预览里的样例值，确认聚合方式和页面筛选没有把范围放大或缩小。正式页面里的核心指标，建议让业务负责人确认口径。

配置完成后，再进入样式和高级配置。样式只负责“怎么显示”，不应该用样式掩盖数据口径问题。

配置面板按“数据、样式、高级”逐步处理。数据页解决字段和过滤；样式页处理标题、颜色、坐标轴、标签；高级页处理联动、跳转、刷新等交互能力。

配置顺序建议固定为：先数据，再样式，最后高级交互。字段未确认前，不宜投入大量时间调整颜色；字段变更后，标题、单位、图例和标签通常也需要同步调整。

完成后可以回到画布查看组件效果。

这张图用于确认“图表已经进入页面”。此时要看三件事：图表标题是否清楚，图表大小是否适合当前区域，右侧配置是否和画布组件对应。页面上有多个图表时，选中组件再改配置，避免改错对象。

图表需要字段配置说明

字段配置完成后，至少做三项检查：

  1. 图表是否有数据。
  2. 指标合计是否与数据集预览或业务样例一致。
  3. 排序、单位、小数位和空值显示是否符合业务习惯。

添加查询组件

查询组件用于让用户在页面上筛选数据，例如日期、区域、部门、产品、项目状态。它通常放在仪表盘顶部。

查询组件不是图表，它的作用是控制其他图表。添加后要在右侧配置目标图表和筛选字段，否则它只是一个没有作用的控件。正式页面通常至少需要日期筛选，经营类页面还会增加区域、门店、产品或项目筛选。

配置流程：

添加查询组件

点击顶部“查询组件”，选择日期、下拉、文本搜索、数字区间等组件，放到页面顶部或筛选区。

绑定目标图表

选中查询组件，在右侧配置要影响哪些图表。一个筛选器可以影响一个或多个图表。

选择筛选字段

给每个目标图表选择对应字段。不同数据集之间字段名称可能不同，要确认业务含义一致。

设置展示方式

日期字段使用日期或日期范围；区域、部门、产品使用下拉或搜索；金额、时长使用数字区间。

设置默认值

常见默认值包括本月、最近 30 天、全部区域、当前用户所属组织。

预览验证

保存后进入预览，切换筛选条件，确认所有目标图表同步变化。

查询组件配置要点：

配置项说明常见问题
目标组件查询组件影响哪些图表漏选图表导致部分图不跟随筛选
筛选字段目标图表中被过滤的字段不同数据集字段含义不一致
展示类型日期、下拉、搜索、区间等字段类型不对，展示类型不可用
默认值页面打开时的筛选范围默认值过窄导致页面像“没数据”
多选是否允许一次选择多个值多选会影响 SQL 和性能
级联上级筛选影响下级选项省份、城市、门店适合级联

页面排版建议

仪表盘不是图表堆叠。排版要围绕阅读顺序组织。

区域推荐内容说明
顶部页面标题、更新时间、查询组件用户先知道看什么、筛什么
第一行核心指标卡3 到 6 个即可，指标越多越难读
中部趋势、对比、结构图解决“变化、差异、构成”问题
底部明细表、口径说明方便追溯和解释

操作建议：

  1. 同一行组件高度尽量一致。
  2. 同类图表使用统一标题、单位和颜色。
  3. 明细表不宜挤压核心趋势图。
  4. 页面标题使用富文本，不依赖第一个图表标题。
  5. 口径说明放在图表附近或页面底部。
  6. 移动端或小屏查看时，优先保证核心指标和筛选器可见。

示例：从零创建一个经营仪表盘

这个示例用于说明创建方法，不是固定模板。可将“销售额、订单数、区域、门店”替换为实际项目中的“需求数、项目、部门、负责人”等字段。

第一步：确定页面结构

建议先做一个简单但完整的结构：

区域示例内容作用
顶部页面标题、日期筛选、区域筛选明确查看范围
第一行销售额、订单数、客单价、毛利率快速判断整体状态
第二行日销售趋势、区域销售排行看变化和差异
第三行渠道结构、门店明细表看构成并追溯明细

新建页面后，先把这些组件放到画布上，再逐个配置数据。初始阶段不宜逐个调整组件颜色，否则容易形成风格不一致的页面。

对应到编辑器中，可以先放一个标题文本、两到四张指标卡、一张趋势图、一张排行图和一张明细表。画布上先有骨架，后续调整会更稳定。

第二步：添加指标卡

指标卡适合放在第一行。示例中可以做四张卡：

指标卡示例指标字段（可替换）聚合方式展示建议
销售额sales_amount求和千分位，单位万元
订单数order_count求和整数
客单价avg_order_amount平均保留 1 到 2 位小数
毛利率gross_margin_rate平均百分比

创建时先添加一张指标卡，配置好标题、单位和字号后复制三份，再分别替换指标字段。这样能保证四张卡的样式一致。

指标卡验收时，不只看数字是否出现，还要确认标题是否包含范围。例如“销售额”不如“本月销售额”清楚；“完成率”不如“本月需求完成率”清楚。

可按以下步骤操作：

添加第一张指标卡

从图表面板选择指标卡，放到页面第一行左侧。选择经营数据集，把“销售额”或自己的核心指标拖到指标位置。

设置标题和单位

标题写成“本月销售额”或“本月交付需求数”。金额统一万元或元，数量统一整数。

复制组件

复制三份指标卡，横向排列，保持相同高度和间距。

替换字段

分别把指标换成订单数、客单价、毛利率，或替换成需求数、完成率、平均周期、风险数。

预览核对

保存并预览，确认筛选条件改变后四张指标卡都同步变化。

第三步：添加趋势图

趋势图回答“最近是否变好”。示例配置如下：

配置项示例字段（可替换）
图表类型折线图
X 轴日期
Y 轴销售额、订单数
排序日期升序
过滤最近 30 天或本月

配置完成后检查两件事：日期是否连续，指标单位是否一致。如果销售额和订单数差异很大，建议拆成两张图，或者使用双轴图并明确左右轴单位。

趋势图建议保留横轴日期和悬浮提示。标签过密时不宜全部显示，否则页面可读性会下降。业务汇报中，趋势图标题要写清统计周期，例如“近 30 天销售额趋势”。

配置趋势图时，日期字段是常见出错点。日期如果被识别成文本，就会按字符串排序，2026-10 可能排在 2026-2 前面。遇到这种情况，回到数据集把字段类型改成日期，或者在 SQL 中转换日期。

第四步：添加排行图和明细表

排行图用于定位贡献最高或风险最高的对象；明细表用于进一步追溯。

图表示例字段（可替换）配置重点
区域销售排行区域 + 销售额销售额降序，限制 Top 10
门店销售排行门店 + 销售额门店名称长时用条形图
渠道结构渠道 + 销售额分类不超过 6 个
门店明细表门店、区域、销售额、订单数、毛利率列宽、排序、导出权限

表格不宜放置过多列。正式页面中，表格的价值是追溯问题，而不是把数据库整张表搬进仪表盘。

明细表放在页面底部更合适。用户先看指标和趋势，发现异常后再下拉查看明细。表格如果允许导出，发布前要确认列权限和导出权限。

表格建议只保留能帮助定位问题的列。例如经营页面保留区域、门店、负责人、销售额、订单数、毛利率；项目页面保留项目、需求编号、阶段、负责人、在库天数、优先级。不应把数据库中的所有字段都放进表格。

第五步：添加查询组件

顶部查询组件建议先做日期和业务范围：

添加日期筛选

放在页面顶部，默认值可以设置为本月或最近 30 天。

添加区域筛选

使用下拉或搜索组件，绑定区域字段。

绑定目标图表

日期筛选通常绑定全部图表；区域筛选绑定趋势图、排行图和明细表。

预览验证

保存后进入预览，切换日期和区域，确认所有目标图表都同步变化。

如果多个图表来自不同数据集，字段名称可以不同，但业务含义必须一致。例如一个数据集叫“区域”，另一个叫“销售大区”，只要口径一致就可以映射。

查询组件验收可按固定步骤执行：保存后进入预览，先记录一个指标卡数值，再切换筛选条件。如果目标图表没有变化，回到编辑器检查目标组件、筛选字段和默认值。

这张图可对照“添加筛选器”的入口。日期、下拉、文本搜索、数字区间都属于查询组件。添加后必须绑定目标图表和字段；如果只把组件放到页面上，用户点击它不会影响任何图表。

示例：日期筛选绑定所有图表的“日期”字段，区域筛选绑定所有图表的“区域”字段。若某张图来自另一个数据集，也要为它单独映射字段。

样式配置

样式配置用于改善阅读体验，不用于修改业务口径。

常用配置：

配置建议
标题标题写业务含义，不写字段名
单位金额统一元、万元或亿元，不混用
小数位百分比、金额、数量分别统一
颜色同类指标保持一致，异常值使用固定警示色
图例系列多时打开图例，系列少时可以关闭
标签数值过密时不宜全部显示
坐标轴趋势图需要清晰时间轴，排名图需要清晰分类轴
空值明确显示为空、0 或不显示

正式页面不要过度使用颜色。一个页面中通常只需要一组主色、一组辅助色和少量警示色。

配置联动

联动用于从一个图表筛选其他图表。例如点击“华东区域”，页面中的趋势图、产品排行和明细表都切换到华东数据。

配置步骤：

  1. 保存仪表盘。
  2. 选中作为触发源的图表。
  3. 在右侧高级配置中打开联动设置。
  4. 选择被影响的目标图表。
  5. 设置源字段和目标字段。
  6. 保存并进入预览。
  7. 点击源图表中的维度值，检查目标图表是否变化。

联动字段要求：

要求说明
业务含义一致源字段和目标字段都代表同一类对象
数据类型一致文本对文本、日期对日期，避免隐式转换
目标图表已使用该字段或能被该字段过滤否则联动不会生效
避免过度联动联动过多会让用户难以理解页面状态

配置跳转和外部参数

跳转用于从总览页进入详情页。外部参数用于从 URL、业务系统或其他页面传入筛选条件。

常见场景：

场景配置方式
点击区域进入区域详情内部跳转到另一个仪表盘，并传递区域字段
点击项目进入项目详情内部跳转，传递项目 ID
点击客户打开 CRM外部 URL，拼接客户 ID
从业务系统嵌入页面URL 参数传入当前组织或用户范围

配置建议：

  1. 先保存当前仪表盘。
  2. 选中源图表或打开顶部外部参数配置。
  3. 明确参数名、来源字段和目标字段。
  4. 预览跳转结果。
  5. 用真实用户权限测试一次。

公开分享链接中不得暴露敏感参数，例如手机号、身份证号、内部客户编号等。

保存、预览和发布

操作含义使用时机
保存保存编辑状态每完成一组组件就保存
预览按用户视角查看页面配置筛选、联动、跳转后必须预览
发布让授权用户访问正式版本验证完成后发布
取消发布暂停正式访问页面下线或口径待修复
复制基于已有页面创建新版本月报、周报、相似部门页面

发布前检查：

  1. 所有图表都有数据。
  2. 查询组件能影响正确图表。
  3. 图表标题、单位、小数位统一。
  4. 页面权限符合目标用户。
  5. 普通账号可以打开页面。
  6. 分享链接或嵌入地址可以正常访问。
  7. 预览状态下没有明显滚动错位、遮挡和空白。

常见问题

问题可能原因处理方法
图表显示空白未选择数据集、字段缺失、筛选过严先检查字段配置，再清空筛选预览
选择不到数据集没有数据集权限或文件夹不对检查数据集授权和资源目录
指标数值不对聚合方式、关联关系或筛选条件错误回到数据集预览核对
日期排序异常日期字段是文本类型在数据集中修正字段类型
查询组件不生效未绑定目标图表或字段映射错误检查目标组件和筛选字段
联动后图表没变化源字段和目标字段含义或类型不一致重新映射字段
普通用户打不开页面、数据集或数据源权限不足用普通账号逐级验证权限
发布后还是旧内容未保存、未发布或浏览器缓存保存并发布后刷新页面
